# ESP32 IPphone

ESP-IDF проект IP-телефона / интеркома на ESP32. Текущая рабочая ветка содержит прототип push-to-talk по ESP-NOW: одна плата записывает голос с I2S-микрофона INMP441 и передает его на другую ESP32, вторая воспроизводит звук через I2S-усилитель MAX98357A.

## Окружение

- ESP-IDF: `v5.5.4`
- Целевая плата: `esp32`
- IDE: Visual Studio Code с расширением Espressif IDF

## Быстрый старт

```powershell
idf.py set-target esp32
idf.py build
idf.py -p COMx flash monitor
```

Одна и та же прошивка заливается на обе ESP32. Для второй платы используйте ее COM-порт.

## Режимы работы

- Короткое нажатие `BOOT`: записать 3 секунды голоса и отправить запись по ESP-NOW.
- Удержание `BOOT` дольше 0.5 секунды: live push-to-talk, передача идет пока кнопка удерживается.
- Приемник воспроизводит входящие ESP-NOW аудиопакеты через MAX98357A.
- Зеленый светодиод горит в режиме ожидания.
- Красный светодиод горит во время передачи или приема голоса.

## Подключение

### INMP441

| INMP441 | ESP32 |
| --- | --- |
| `VDD` | `3V3` |
| `GND` | `GND` |
| `SCK` | `GPIO18` |
| `WS` | `GPIO19` |
| `SD` | `GPIO23` |
| `L/R` | `GND` |

### MAX98357A

| MAX98357A | ESP32 |
| --- | --- |
| `VIN` | `5V` или `3V3` |
| `GND` | `GND` |
| `BCLK` | `GPIO5` |
| `LRC` / `LRCLK` | `GPIO17` |
| `DIN` | `GPIO22` |
| `SD` / `SHDN` | `3V3` или не подключать, если модуль уже подтянут |
| `GAIN` | не подключать |

Динамик подключается только к `OUT+` и `OUT-` усилителя MAX98357A.

### Светодиоды

По умолчанию светодиоды подключаются по схеме `GPIO -> резистор -> LED -> GND`, активный уровень `1`.

| Назначение | ESP32 |
| --- | --- |
| Зеленый, ожидание | `GPIO2` |
| Красный, TX/RX голос | `GPIO16` |

Если светодиод подключен к `3V3` и включается уровнем `0`, поменяйте active level через `idf.py menuconfig`.

## Настройки GPIO

Пины вынесены в:

```text
Component config -> ESP32 IPphone
```

Доступные группы:

- `Keypad pins`
- `I2S audio pins`
- `Display pins`
- `Status LED pins`

Значения по умолчанию также лежат в `sdkconfig.defaults`.

## Структура

- `main/` - точка входа, обработка BOOT и общего сценария.
- `components/audio/` - I2S микрофон, I2S динамик, запись и воспроизведение PCM.
- `components/network_audio/` - ESP-NOW передача и прием голосовых пакетов.
- `components/status_leds/` - зеленый/красный индикаторы состояния.
- `components/keypad/` - заготовка для матричной клавиатуры.
- `components/display/` - заготовка для дисплея.

## Что отправлять в Git

Отправлять:

- исходники `main/` и `components/`;
- `CMakeLists.txt`;
- `sdkconfig.defaults`;
- `.gitignore`;
- `README.md`.

Не отправлять:

- `build/`;
- `sdkconfig`;
- `sdkconfig.old`;
- локальные настройки порта в `.vscode/settings.json`, если менялся только `idf.portWin`;
- временные файлы IDE и ОС.

## Проверка

1. Прошейте обе ESP32 одной прошивкой.
2. На обеих платах в ожидании должен гореть зеленый светодиод.
3. Нажмите и удерживайте `BOOT` на передающей плате.
4. После короткого звукового сигнала говорите в микрофон.
5. На обеих платах во время передачи/приема должен гореть красный светодиод.
6. После отпускания `BOOT` платы должны вернуться в ожидание, снова горит зеленый.

Если live PTT работает нестабильно, проверьте короткое нажатие `BOOT`: оно использует старый рабочий режим записи 3 секунд и помогает отделить проблему ESP-NOW потока от проблемы аудио.
