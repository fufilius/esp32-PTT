# ESP32 IPphone

ESP-IDF проект IP-телефона / интеркома на ESP32. Текущая ветка содержит рабочий прототип:

- live push-to-talk по ESP-NOW;
- запись голоса на microSD в WAV-файл;
- воспроизведение записанного WAV через MAX98357A;
- индикация состояния зеленым и красным светодиодами.

## Окружение

- ESP-IDF: `v5.5.4`
- Целевая плата: `esp32`
- IDE: Visual Studio Code с расширением Espressif IDF

## Быстрый старт

```powershell
idf.py set-target esp32
idf.py build
idf.py -p COMx flash monitor
```

Одна и та же прошивка заливается на обе ESP32. Для второй платы используйте ее COM-порт.

## Режимы основной кнопки

- Короткое нажатие, если запись не идет: начать запись голоса на microSD, красный светодиод включается.
- Короткое нажатие, если запись уже идет: остановить запись, сохранить `/sdcard/record.wav`, затем воспроизвести запись в динамик.
- Удержание дольше 0.5 секунды: live PTT по ESP-NOW, передача идет пока кнопка удерживается.

Звуковые сигналы отключены: состояние показывают светодиоды.

По умолчанию используется встроенная кнопка `BOOT` на `GPIO0`. Позже ее можно заменить обычной кнопкой через `idf.py menuconfig`.

Антидребезг основной кнопки:

- debounce: `40 ms`;
- активный уровень: `0`;
- схема подключения обычной кнопки: `GPIO -> кнопка -> GND`;
- внутренняя подтяжка ESP32 включается автоматически.

## Индикация

- Зеленый светодиод: режим ожидания.
- Красный светодиод: запись на microSD, воспроизведение записи, ESP-NOW передача или ESP-NOW прием.

## Подключение

### INMP441

| INMP441 | ESP32 |
| --- | --- |
| `VDD` | `3V3` |
| `GND` | `GND` |
| `SCK` | `GPIO18` |
| `WS` | `GPIO19` |
| `SD` | `GPIO23` |
| `L/R` | `GND` |

### MAX98357A

| MAX98357A | ESP32 |
| --- | --- |
| `VIN` | `5V` или `3V3` |
| `GND` | `GND` |
| `BCLK` | `GPIO5` |
| `LRC` / `LRCLK` | `GPIO17` |
| `DIN` | `GPIO22` |
| `SD` / `SHDN` | `3V3` или не подключать, если модуль уже подтянут |
| `GAIN` | не подключать |

Динамик подключается только к `OUT+` и `OUT-` усилителя MAX98357A.

### microSD SPI

| microSD | ESP32 |
| --- | --- |
| `VCC` | `3V3` |
| `GND` | `GND` |
| `SCK` | `GPIO14` |
| `MOSI` | `GPIO13` |
| `MISO` | `GPIO27` |
| `CS` | `GPIO32` |

Эти GPIO раньше были в заготовке под клавиатуру. В текущей логике клавиатура не инициализируется, чтобы не конфликтовать с microSD.

### Обычная кнопка

| Кнопка | ESP32 |
| --- | --- |
| Контакт 1 | `GPIO0` по умолчанию, можно поменять |
| Контакт 2 | `GND` |

Для длинных проводов рекомендуется внешний pull-up `10 кОм` к `3V3`.

### Светодиоды

По умолчанию светодиоды подключаются по схеме `GPIO -> резистор -> LED -> GND`, активный уровень `1`.

| Назначение | ESP32 |
| --- | --- |
| Зеленый, ожидание | `GPIO2` |
| Красный, активный голосовой режим | `GPIO16` |

Если светодиод подключен к `3V3` и включается уровнем `0`, поменяйте active level через `idf.py menuconfig`.

## Настройки GPIO

Пины вынесены в:

```text
Component config -> ESP32 IPphone
```

Доступные группы:

- `Main button`
- `I2S audio pins`
- `Display pins`
- `Status LED pins`
- `microSD SPI pins`

Значения по умолчанию также лежат в `sdkconfig.defaults`.

## Структура

- `main/` - точка входа и общий сценарий.
- `components/button/` - основная кнопка, антидребезг, click/hold события.
- `components/audio/` - I2S микрофон, I2S динамик, запись и воспроизведение PCM.
- `components/network_audio/` - ESP-NOW передача и прием голосовых пакетов.
- `components/storage/` - mount microSD, запись и чтение WAV.
- `components/status_leds/` - зеленый/красный индикаторы состояния.
- `components/keypad/` - заготовка для матричной клавиатуры, сейчас не используется в `app_main`.
- `components/display/` - заготовка для дисплея.

## Проверка microSD

1. Вставьте microSD с FAT/FAT32.
2. Прошейте ESP32.
3. В ожидании должен гореть зеленый светодиод.
4. Коротко нажмите основную кнопку: загорится красный, начнется запись.
5. Говорите в микрофон.
6. Еще раз коротко нажмите основную кнопку: запись остановится и сразу проиграется через динамик.
7. На карте появится файл `/record.wav`.

## Проверка PTT

1. Прошейте обе ESP32 одной прошивкой.
2. Нажмите и удерживайте основную кнопку на передающей плате.
3. Говорите в микрофон, пока держите кнопку.
4. На передающей и принимающей платах во время передачи/приема должен гореть красный светодиод.
5. После отпускания кнопки платы возвращаются в ожидание, снова горит зеленый.

## Что отправлять в Git

Отправлять:

- исходники `main/` и `components/`;
- `CMakeLists.txt`;
- `sdkconfig.defaults`;
- `.gitignore`;
- `README.md`.

Не отправлять:

- `build/`;
- `sdkconfig`;
- `sdkconfig.old`;
- локальные настройки порта в `.vscode/settings.json`, если менялся только `idf.portWin`;
- временные файлы IDE и ОС.
